[] Стиль кода: Camel Style
[] Фреймворк: ASP.NET Angular
[] Хост: -
[] Git control: 
    + каждой задаче полагается отдельная ветка на базе мастера
    + после завершения работы над задачей ,перед попаданием задачи в "Ready for review", создается пул реквест
    + после проверки пул реквеста ветка задачи подлежит мерджу с удалением самой ветки
    + ветка любой задачи должна быть привязана к задаче непосредственно
    + задачи не содержащие код игнорируют промежуточные столбцы между "In Progress" и "Done" т.е. не подлежат проверке
